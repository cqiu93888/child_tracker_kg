# 追蹤狀況診斷與改善

當輸出影片追蹤很差、幾乎沒有顯示已辨識名字時，可依下列步驟排查與調整。

---

## 一、先跑診斷（找出問題點）

在專案目錄執行：

```bash
python diagnose_tracking.py --video data/input/test2.mp4 --frame 30 --save data/output/diagnose.png
```

- `--video`：你的**輸入影片**路徑  
- `--frame`：檢查第幾幀（預設 30，可改成 0 或中間某幀）  
- `--save`：可選，會輸出一張該幀的偵測結果圖（臉框畫在畫面上）

**看終端輸出：**

1. **YOLO 偵測到人體框數**  
   - 若是 0：影片該幀可能沒人、或 YOLO 沒偵測到，可換一幀或確認畫面中有人。
2. **臉部 預設 / upsample=2 / upsample=3 偵測到臉數**  
   - 若都是 0：該幀臉部偵測不到（臉太小、角度、遮擋）。可換一幀或確認臉有朝鏡頭。  
   - 若 **YOLO 人數 > 臉數**：代表有人沒被偵測到臉，診斷會提示可試 upsample=3 或換幀；側臉、遮擋、表情大時易漏。
3. **臉與註冊者最近距離 / 符合門檻的臉數**  
   - 若「符合門檻的臉數」為 0：代表有偵測到臉，但都和註冊名單對不上。  
   - 看「臉與註冊者最近距離」：若普遍 > 0.6，表示影片臉與註冊照差異大，需要**放寬門檻**或**改善註冊照**。

依診斷結果對應下面「二、常見問題與解法」。

---

## 二、常見問題與解法

### 1. 有臉但都未匹配到名字（診斷顯示符合門檻=0）

**原因：** 影片中光線、角度、表情與註冊照差異大，face_distance 超過目前門檻。

**作法：**

- 在 **`config.py`** 調大 `FACE_MATCH_TOLERANCE`：  
  - 原本 0.55 → 試 0.62～0.68（愈大愈寬鬆，辨識出名字機率愈高，但認錯人機率也略增）。
- 重新跑一次診斷，看「臉與註冊者最近距離」是否小於新的門檻。
- 再跑一次影片處理：  
  `python main.py process --video data/input/test2.mp4 --output data/output/output.mp4`

### 2. 臉部偵測數為 0（診斷顯示偵測到臉數=0）

**原因：** 該幀臉太小、側面、或被擋住，預設偵測不到。

**作法：**

- 程式已改為使用 **number_of_times_to_upsample=3**（加強小臉偵測，與實際追蹤一致），若仍為 0：
  - 換一幀再跑診斷（例如 `--frame 50`）。
  - 確認**輸入影片**中該段有清楚、朝鏡頭的臉。
- 若影片解析度很低，可考慮先將影片放大再處理（或換較高解析度來源）。

### 3. YOLO 人體框數為 0

**原因：** 該幀沒有人、或人體被裁切/過小，YOLO 沒偵測到。

**作法：**

- 換一幀（例如有人站或坐清楚的幀）再跑診斷。
- 確認 **input 影片** 在該時間點畫面中確實有人。

### 4. 追蹤 ID 一直跳、同一個人編號變來變去

**原因：** 幀與幀之間人框位置變化大，IoU 過低導致被當成新的人。

**作法：**

- 程式已加入「中心距離」備援：當 IoU 不夠時，若中心距離 < 200px 且 IoU > 0.05 仍視為同一人。
- 若仍不穩，可檢查 **input 影片** 是否晃動很大、或幀率很低，考慮先穩定影片或提高幀率。

### 5. YOLO 偵測到 N 人但只偵測到 M 張臉（N > M）

**原因：** 部分人臉因側臉、遮擋、表情大或臉較小，未被臉部偵測抓到。

**作法：**

- 診斷腳本已使用 **upsample=3**（與實際 process 相同），會顯示三種偵測的臉數，並取最多者。
- 若仍不足：可換不同幀（`--frame 0`、`50`、`100`）多跑幾次診斷，找臉數較多的幀；或接受部分幀漏偵，靠追蹤延續顯示「未知-N」。

### 6. 註冊照與影片差異大

**建議：**

- 註冊照盡量用**正面、光線充足、表情自然**的清晰照。
- 若只有側面或模糊照，可補拍一張正面照重新註冊該員，再跑一次 process。

---

## 三、已做的程式改善（本次）

- **臉部辨識**：改用 **face_distance + 最佳匹配**（取距離最小的註冊者，且距離 < 門檻才採用），比單純 compare_faces 更穩定。
- **小臉偵測**：臉部偵測改為 **number_of_times_to_upsample=3**，較容易偵測到幼兒坐姿時較小的臉；診斷腳本也會顯示 upsample=2 / 3 的結果並給建議。
- **人框與臉的對應**：人體框往上擴 20% 再檢查「臉中心是否在人框內」，坐姿時頭在框上方也能對應到。
- **臉–人對應門檻**：臉與人框重疊分數從 0.05 降到 0.03，較容易把人與臉綁在一起。
- **辨識門檻**：影片辨識時使用較寬的 tolerance（約 0.68 以內），提高對註冊名單的匹配率。
- **ID 延續**：當 IoU 不夠時，用「中心距離 < 200px」作為同一人的備援，減少 ID 閃爍。

---

## 四、建議操作順序

1. 跑一次診斷：  
   `python diagnose_tracking.py --video data/input/test2.mp4 --frame 30 --save data/output/diagnose.png`
2. 看終端與（若有）`data/output/diagnose.png`，對照「二」調整 config 或註冊/影片。
3. 再跑影片處理：  
   `python main.py process --video data/input/test2.mp4 --output data/output/output.mp4`
4. 若仍不理想，可把診斷終端輸出（以及若可描述：哪一段、幾個人、坐或站）提供給開發者，方便進一步對症修改。
